Detailed Description

Win32Signature is for __stdcall calling convention: parameters pushed right-to-left, return value in eax register.

Win32TcSignature is for "thiscall" signatures, i.e. those that have register ecx as the first parameter. Only needs to override a few member functions; the rest can inherit from Win32Signature
