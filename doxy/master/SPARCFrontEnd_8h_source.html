70  /**
71  * Determines if the given call and delay instruction consitute a call
72  * where callee returns to the caller's caller. That is:
73  * ProcA | ProcB: | ProcC:
74  * -------------|---------------|---------------
75  * ... | ... | ...
76  * call ProcB | call ProcC | ret
77  * ... | restore | ...
78  *
79  * The restore instruction in ProcB will effectively set %o7 to be %i7,
80  * the address to which ProcB will return. So in effect ProcC will return
81  * to ProcA at the position just after the call to ProcB. This is equivalent
82  * to ProcC returning to ProcB which then immediately returns to ProcA.
83  *
84  * \note We don't set a label at the return, because we also have to
85  * force a jump at the call BB, and in some cases we don't
86  * have the call BB as yet. So these two are up to the caller
87  * \note The name of this function is now somewhat irrelevant.
88  * The whole function is somewhat irrelevant; it dates from
89  * the times when you would always find an actual restore
90  * in the delay slot. With some patterns, this is no longer true.
91  *
92  * \param call the RTL for the caller (e.g. "call ProcC" above)
93  * \param rtl pointer to the RTL for the call instruction
94  * \param delay the RTL for the delay instruction (e.g. "restore")
95  * \returns The basic block containing the single return instruction
96  * if this optimisation applies, nullptr otherwise.
97  */
98  BasicBlock *optimizeCallReturn(CallStatement *call, const RTL *rtl, const RTL *delay,
99  UserProc *proc);
