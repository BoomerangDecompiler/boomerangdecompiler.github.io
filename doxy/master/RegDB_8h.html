Process the effects of overlapped registers for stmt. Example: (x86 register overlap) Suppose stmt is ax := 0x1234, and eax and ah are used in the procedure that contains stmt. Then we need to generate two additional statements: 32 eax := (eax & 0xFFFF0000) | ax, and 8 ah := ax@[0..7] to model the effects of overlapped registers. This procedure returns the additional statements in a new RTL. We do not need to generate an assignment for al, since al is not used in the current procedure (This is indicated by usedRegs not containing al).

Returns
all additional statements

Emit a new statement assigning the content of rhs into lhs. There are 2 cases:

  1. The LHS is larger. In this case, assign only the bits of lhs that also belong to rhs (e.g. eax := (eax & 0xFFFF00FF) | (ah << 8)) (Note: It might also be possible to emit eax@[8:15] := ah, but other code does not support this yet).
  2. The RHS is larger. In this case, use only the bits of the RHS that also belong to lhs. (e.g. ah := eax@[8..15])
