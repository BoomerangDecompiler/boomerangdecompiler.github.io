86  /**
87  * Create a new Basic Block for this CFG.
88  * If the BB is blocked by a larger complete BB, the existing BB will be split at the first
89  * address of \p bbRTLs; in this case this function returns nullptr (since no BB was created).
90  * The case of the new BB being blocked by a smaller complete BB is not handled by this method;
91  * use \ref ProcCFG::ensureBBExists instead.
92  *
93  * The new BB might also be blocked by exising incomplete BBs.
94  * If this is the case, the new BB will be split at all blocking incomplete BBs,
95  * and fallthrough edges will be added between parts of the split BB.
96  * In this case, the incomplete BBs will be removed (since we just completed them).
97  *
98  * \param bbType Type of the new Basic Block
99  * \param bbRTLs RTL list with semantics of all instructions contained in this BB.
100  * Must not be empty.
101  *
102  * \returns the newly created BB, or the exisitng BB if the new BB is the same as
103  * another exising complete BB.
104  */
105  BasicBlock *createBB(BBType bbType, std::unique_ptr<RTLList> bbRTLs);

114  /**

116  *
117  * Explicit labels are addresses that have already been tagged as being labels

120  * In this case, the existing complete BB is split. If \p currBB is the BB that gets split,

124  * \param currBB See above

213  /**
214  * Split \p bb into a "low" and "high" part at the RTL associated with \p splitAddr.
215  * The type of the "low" BB becomes fall-through. The type of the "high" part becomes the type
216  * of \p bb.
217  *
218  * \ | / \ | /
219  * +---+ bb +---+ BB1
220  * | | +---+
221  * | | ==> | Fallthrough
222  * +---+ +---+
223  * / | \ +---+ BB2
224  * / | \
225  *
226  * If \p splitAddr is not in the range [bb->getLowAddr, bb->getHiAddr], the split fails.
227  * \param bb pointer to the BB to be split
228  * \param splitAddr address of RTL to become the start of the new BB
229  * \param newBB if non zero, it remains as the "bottom" part of the BB, and splitBB only
230  * modifies the top part to not overlap. If this is the case, the RTLs of the original BB are
231  * deleted. \returns If the merge is successful, returns the "high" part of the split BB.
232  * Otherwise, returns the original BB.
233  */
234  BasicBlock *splitBB(BasicBlock *bb, Address splitAddr, BasicBlock *newBB = nullptr);
