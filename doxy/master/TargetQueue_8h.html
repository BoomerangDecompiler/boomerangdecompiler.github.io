Visit a destination as a label, i.e. check whether we need to queue it as a new BB to create later.

Note
at present, it is important to visit an address BEFORE an out edge is added to that address. This is because adding an out edge enters the address into the ProcCFG's BB map, and it looks like the BB has already been visited, and it gets overlooked. It would be better to have a scheme whereby the order of calling these functions (i.e. visit() and AddOutEdge()) did not matter.
