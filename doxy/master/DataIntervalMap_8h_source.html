39 /**
40  * This class is used to represent local variables in procedures, and global variables for the
41  * program.
42  *
43  * The concept is that the data space (the current procedure's stack or the global data space) has
44  * to be partitioned into separate variables of various sizes and types. If a new variable is
45  * inserted that would cause an overlap, the types have to be reconciled such that they no longer
46  * conflict (generally, the smaller type becomes a member of the larger type, which has to be a
47  * structure or an array). Each procedure and the Prog object have a map from Address (stack offset
48  * from sp{0} for locals, or native address for globals), to an object of this class. A multimap is
49  * not needed, as the type of the entry specifies the overlapping.
50  */
51 class BOOMERANG_API DataIntervalMap
