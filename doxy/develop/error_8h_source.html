56 /* NX_DURING, NX_HANDLER and NX_ENDHANDLER are always used like:
57  *
58  * NX_DURING
59  * some code which might raise an error
60  * NX_HANDLER
61  * code that will be jumped to if an error occurs
62  * NX_ENDHANDLER
63  *
64  * If any error is raised within the first block of code, the second block
65  * of code will be jumped to. Typically, this code will clean up any
66  * resources allocated in the routine, possibly case on the error code
67  * and perform special processing, and default to RERAISE the error to
68  * the next handler. Within the scope of the handler, a local variable
69  * called NXLocalHandler of type NXHandler holds information about the
70  * error raised.
71  *
72  * It is illegal to exit the first block of code by any other means than
73  * NX_VALRETURN, NX_VOIDRETURN, or just falling out the bottom.
74  */
