The ExpModifier class is used to iterate over all subexpressions in an expression.

It contains methods for each kind of subexpression found in an and can be used to eliminate switch statements. It is a little more expensive to use than ExpVisitor, but can make changes to the expression

Change the expression before visiting children. The default behaviour is to not modify the expression and to recurse to all child expressions.

Note
preModify() functions must not change the type of the expression, e.g. from Binary to Const
Parameters
[in]expthe expression to change
[out]visitChildrentrue to continue visiting children.
Returns
The modified expression (must not be null). Note that this is not necessarily the same expression as exp, but an expression of the same type as exp.
