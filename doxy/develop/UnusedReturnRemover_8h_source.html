22 class UnusedReturnRemover
23 {
24 public:
25  explicit UnusedReturnRemover(Prog *prog);

27 public:
28  /**
29  * Remove unused return locations.
30  * This is the global removing of unused and redundant returns. The initial idea
31  * is simple enough: remove some returns according to the formula:
32  * returns(p) = modifieds(p) isect union(live at c) for all c calling p.
33  *
34  * However, removing returns reduces the uses, leading to three effects:
35  * 1) The statement that defines the return, if only used by that return, becomes unused
36  * 2) if the return is implicitly defined, then the parameters may be reduced, which affects all
37  * callers 3) if the return is defined at a call, the location may no longer be live at the
38  * call. If not, you need to check the child, and do the union again (hence needing a list of
39  * callers) to find out if this change also affects that child. \returns true if any change
40  */
41  bool removeUnusedReturns();
42 
43 private:
44  /**
45  * Remove any returns that are not used by any callers
46  *
47  * Remove unused returns for this procedure, based on the equation:
48  * returns = modifieds isect union(live at c) for all c calling this procedure.
49  * The intersection operation will only remove locations. Removing returns can have three
50  * effects for each component y used by that return (e.g. if return r24 := r25{10} + r26{20} is
51  * removed, statements 10 and 20 will be affected and y will take the values r25{10} and
52  * r26{20}): 1) a statement s defining a return becomes unused if the only use of its definition
53  * was y 2) a call statement c defining y will no longer have y live if the return was the only
54  * use of y. This could cause a change to the returns of c's destination, so
55  * removeRedundantReturns has to be called for c's destination proc (if it turns out to be the
56  * only definition, and that proc was not already scheduled for return removing). 3) if y is a
57  * parameter (i.e. y is of the form loc{0}), then the signature of this procedure changes, and
58  * all callers have to have their arguments trimmed, and a similar process has to be applied to
59  * all those caller's removed arguments as is applied here to the removed returns.
60  *
61  * The \a removeRetSet is the set of procedures to process with this logic; caller in Prog calls
62  * all elements in this set (only add procs to this set, never remove)
63  *
64  * \returns true if any change
65  */
66  bool removeUnusedParamsAndReturns(UserProc *proc);
67 
68  /**
69  * Update parameters and call livenesses to take into account the changes
70  * caused by removing a return from this procedure, or a callee's parameter
71  * (which affects this procedure's arguments, which are also uses).
72  *
73  * Need to save the old parameters and call livenesses, redo the dataflow and
74  * removal of unused statements, recalculate the parameters and call livenesses,
75  * and if either or both of these are changed, recurse to parents or those calls'
76  * children respectively. (When call livenesses change like this, it means that
77  * the recently removed return was the only use of that liveness, i.e. there was a
78  * return chain.)
79  * \sa removeRedundantReturns().
80  */
81  void updateForUseChange(UserProc *proc);
82 
83  /// Remove returns from the return statement to match the signature of \p proc
84  /// \returns true if any change
85  bool removeReturnsToMatchSignature(UserProc *proc);
86 
87 private:
88  Prog *m_prog;
89  ProcSet m_removeRetSet; ///< UserProcs that need their returns updated
90 };
