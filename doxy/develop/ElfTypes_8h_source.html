326 /**
327  * Abbreviations for relocations:
328  * A The addend used to compute the value of the relocatable field.
329  * B The base address at which a shared object is loaded into memory during execution.
330  * Generally, a shared object file is built with a base virtual address of 0.
331  * However, the execution address of the shared object is different. See
332  * “Program Header” on page 282.
333  * G The offset into the global offset table at which the address of the relocation entry's
334  * symbol resides during execution. See “Global Offset Table (Processor-Specific)” on
335  * page 309.
336  * GOT The address of the global offset table. See “Global Offset Table (Processor-Specific)”
337  * on page 309.
338  * L The section offset or address of the procedure linkage table entry for a symbol. See
339  * “Procedure Linkage Table (Processor-Specific)” on page 310.
340  * O The secondary addend used to compute the value of the relocation field. This
341  * addend is extracted from the r_info field by applying the ELF64_R_TYPE_DATA macro.
342  * (64-bit SPARC only)
343  * P The section offset or address of the storage unit being relocated, computed using
344  * r_offset.
345  * S The value of the symbol whose index resides in the relocation entry.
346  * Z The size of the symbol whose index resides in the relocation entry.
347  */
